# [2.02] RDD, DataFrame, Dataset API の歴史と Catalyst Optimizer

Sparkには複数の書き方（APIレベル）があります。

### 1. RDD (Resilient Distributed Dataset)
*   **第一世代**: 低レベルなデータ表現。
*   **特徴**: 自由自在だが、Spark側で「最適化」がしにくい。現在はエンジニアが直接書くことは稀です。

### 2. DataFrame (データフレーム)
*   **第二世代**: テーブル形式（行と列）のデータ表現。
*   **特徴**: **もっとも実務で使われる。** 内部で「Catalyst Optimizer」が「どうすれば世界一効率よくJOINできるか？」を自動計算してくれます。

### 3. Dataset
*   **第三世代**: 型定義（Java/Scalaのクラス）を組み込んだ DataFrame。
*   **特徴**: Python (PySpark) ではサポートされておらず、事実上 Scala/Java 向けの機能です。

### 🌟 Catalyst Optimizer (カタリスト・オプティマイザ)
私たちが書いたSQLやPythonコードを読み、論理的に「このフィルタリングをJOINの前に持ってきたほうが速いぞ」といった最適化を行うSparkの頭脳です。
