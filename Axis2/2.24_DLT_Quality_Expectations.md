# [2.24] DLT Expectations：パイプライン内で「品質」を保証する

「ゴミが混ざったら止める」か、「ゴミだけ隔離する」か。

## 1. Expectations とは
DLT 独自の強力なデータ品質チェック機能です。
- **仕組み**: 各テーブルの定義に、守るべきルール（例： `id IS NOT NULL`）を記述します。

## 2. 3つのアクションレベル
1.  **expect**: ルール違反があってもレコードを通す。ただし、ダッシュボードに「違反率」を表示する。
2.  **expect_or_drop**: ルール違反のレコードは **捨てて（ドロップして）**、正常なものだけを次の層へ渡す。
3.  **expect_or_fail**: 1件でも違反があれば、**パイプライン全体を即座に停止** させる。

## 3. 実装例
```python
@dlt.table
@dlt.expect_or_drop("valid_email", "email LIKE '%@%'")
@dlt.expect_or_fail("has_id", "id IS NOT NULL")
def cleaned_users():
  return dlt.read("raw_users")
```

---
## 🚨 The Incident: 意図しないデータ全消失
**事象**: `expect_or_fail` を設定していたところ、ソースシステムの不具合で一瞬だけ「全データの ID が NULL」になり、本番パイプラインが深夜に停止、翌朝のレポートが真っ白になった。
**解決策**: 最初は `expect`（警告のみ）で様子を見、データの傾向を掴んでから、絶対に許容できない項目のみ `or_drop` や `or_fail` に格上げする、段階的な導入が実務では安全です。
