# [2.24] DLT Expectations：データ品質チェック（DROP/FAIL制約）

「データの品質が悪ければ、処理を止めるか、その行だけ捨てたい」
この要件をコード1行で実装できるのが **Expectations** です。

### 1. 3つのアクション
1.  **expect ... (通知のみ)**: 品質ルールを満たさないデータがあっても、記録は残すが処理は止まらない。
2.  **expect_or_drop ... (行を捨てる)**: その行だけをバイパスし、正常なデータだけで処理を続ける。
3.  **expect_or_fail ... (処理中断)**: 不正データがあれば、即座にJobをエラーにする。

### 2. 実装イメージ
```python
@dlt.expect_or_drop("valid_revenue", "revenue > 0")
@dlt.table
def cleaned_sales():
    return dlt.read("raw_sales")
```

---
**💡 運用設計のポイント:**
「異常データが何件あったか（期待値チェックに引っかかったか）」という統計は、DLTダッシュボードで一目で確認可能です。これにより、データ品質の劣化（Data Drift）の予兆を早期に発見できます。
