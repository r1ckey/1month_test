# [2.13] OPTIMIZE と Small File Problem：ファイル結合による高速化

「1KBのファイルが100万個ある」状態は、データレイクにとって最悪の毒です。

## 1. Small File Problem とは
データの書き込みを繰り返すと、中身のスカスカな小さい Parquet ファイルが大量に生成されます。
- **なぜ遅い？**: Sparkが各ファイルを開くたびに「ファイルを開いて閉じる」というメタデータ操作のオーバーヘッドが発生し、読み込みが激重になります。

## 2. OPTIMIZE による「 compaction（圧縮）」
バラバラになった小さなファイルを、理想的なサイズ（デフォルト約1GB）に結合して書き直します。
```sql
OPTIMIZE main.sales.orders;
```
たったこれだけで、読み込み速度が 10倍 以上改善することもあります。

## 3. 実務での自動化（Auto-Optimize）
毎回手で叩くのは面倒なため、テーブルのプロパティで自動化を検討します。
- `delta.autoOptimize.optimizeWrite`: 書き込み時に少し賢くファイルをまとめる。
- `delta.autoOptimize.autoCompact`: 書き込み直後に小さすぎるファイルを裏でまとめる。

---
## 🚨 The Incident: 更新が止まる！？
**事象**: 巨大なテーブルに対して `OPTIMIZE` を実行したところ、その間、他のJobが書き込みできず待たされてしまった。
**対策**: `OPTIMIZE` は重い処理です。深夜のバッチ終了後など、アクセスの少ない時間帯に定期実行するスケジュールを組みましょう。
