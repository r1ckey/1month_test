# [2.25] 変更データキャプチャ（CDC）: SCD Type1/2の実装設計

業務システムDB（SQL Serverなど）の「変化」を同期し続ける手法です。

### 1. SCD Type 1 (上書き型)
「メールアドレスが変更されたら、古い方は消して新しい方で上書きする」形式。最新の状態だけ管理。

### 2. SCD Type 2 (履歴保持型)
「Aさんは2023年までは東京在住、2024年からは横浜在住」という **履歴を全て残す** 形式。

### 🌟 DLT `APPLY CHANGES INTO`
この複雑な履歴管理のロジックを、Databricksでは専用のコマンド1つで行えます。内部で「削除されたデータ」「追加されたデータ」を自動的に判別し、テーブルの履歴を最新化してくれます。
