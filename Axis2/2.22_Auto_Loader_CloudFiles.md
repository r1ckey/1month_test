# [2.22] Auto Loader (CloudFiles) とスキーマ進化の自動追随

「ADLS にファイルが置かれたら即座に、二重に読み込むことなく、かつ安全に」読み込むための最強ツール。

## 1. Auto Loader (CloudFiles) とは
Databricks 独自の機能で、数万〜数百万個のファイルが格納されているフォルダから、新着分だけを効率的に見つけ出す仕組みです。
- **仕組み**: 内部的には Azure Queue Storage と Event Grid を使い、ファイル到着の通知を Spark が受け取ります。
- **メリット**: `spark.read` をループで回すよりも圧倒的に速く、ファイル数が 1億 個を超えても性能が落ちません。

## 2. スキーマ進化 (Schema Evolution)
ソースシステムの変更で、CSVに突然「新しい列」が増えた場合でも、Jobを止めずに取り込むことができます。
- `cloudFiles.schemaLocation`: スキーマの履歴を保存するパス。
- `cloudFiles.schemaEvolutionMode`: `addNewColumns`（新しい列があれば自動で追加する）などが指定可能。

## 3. 実装例
```python
df = spark.readStream.format("cloudFiles") \
  .option("cloudFiles.format", "csv") \
  .option("cloudFiles.schemaLocation", "/tmp/checkpoints/_schema") \
  .load("abfss://raw-data@storage.dfs.core.windows.net/orders")
```

---
## 🚨 The Incident: 型の不一致で全停止
**事象**: スキーマ進化は「新しい列の追加」には強いですが、**「既存の列の型変更（StringがIntになった等）」** が起きるとエラーで止まります。
**解決策**: Bronze層（Axis 2.17）で学んだ通り、Auto Loader でも原則として `cloudFiles.inferColumnTypes` を `false` にし、全てを一旦 `STRING` で受けるのが、実務で「絶対に止まらない」ための秘訣です。
