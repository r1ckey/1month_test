# [2.12] タイムトラベル（Time Travel）を活用した本番トラブル復旧術

「間違えて全行消してしまった！」その絶望を、一行のSQLで「なかったこと」にします。

## 1. タイムトラベルとは
Delta Lake は過去のバージョンの Parquet ファイルを物理的に一定期間残しています。
- `VERSION AS OF`: バージョン番号（0, 1, 2...）で指定。
- `TIMESTAMP AS OF`: 特定の時刻（30分前、昨日の昼など）で指定。

## 2. 実務での「最強のリカバリコード」
間違えてテーブルを壊してしまった時、このコマンド一発で過去の状態に戻せます。
```sql
-- テーブル全体を バージョン 123 の状態に巻き戻す（リストア）
RESTORE TABLE main.sales.orders TO VERSION AS OF 123;
```
Pythonの場合：
```python
df_old = spark.read.option("versionAsOf", 10).table("my_table")
```

## 3. 保存期間の設計
デフォルトでは **30日間** の履歴が保持されます。
- **注意**: 履歴を残せば残すほど、ストレージ（ADLS）の課金が増えます。
- **運用**: 重要な監査データは長めに、使い捨ての中間テーブルは短めに設定を調整するのがアーキテクトの仕事です。

---
## 🚨 The Incident: タイムトラベルができない！？
**事象**: 3日前の状態に戻そうとしたが、`Version not found` エラーが発生した。
**原因**: 前日に `VACUUM` コマンドを実行してしまい、古い物理ファイルが削除されていた。
**教訓**: `VACUUM` は掃除に便利ですが、実行するとそれ以前のタイムトラベルができなくなります。安易に実行しない、または保持期間（Retention）の設定を慎重に行う必要があります。
