# [2.07] データ・スキュー (Data Skew) 検知と Saltキー分散術

「1つのタスク（Worker）だけがいつまでも終わらない」
これがデータエンジニアリングにおける最大の敵「スキュー」です。

## 1. データ・スキューとは？
特定のキー（例：店舗ID「東京店」）にデータが極端に偏っている状態。
- **現象**: 200個のタスクのうち、199個は10秒で終わるのに、1つのタスク（東京店担当）だけが1時間経っても終わらない。

## 2. 検知方法：Spark UI の確認
Spark UI の `Stages` タブで、タスクの実行時間の **「Max」と「Median（中央値）」** を見比べます。
- **スキューの兆候**: Median が 10秒 なのに Max が 30分 だった場合、間違いなく特定のキーにデータが偏っています。

## 3. 解決策：Salting (塩を振る)
偏っているキーにランダムな数値（0〜9など）を付与して一時的に分散させ、集計した後に再度まとめる手法です。
```python
# Saltキーの付与例
from pyspark.sql.functions import lit, rand, concat

# キーを分散させる
df_skewed = df.withColumn("salt", (rand() * 10).cast("int"))
df_skewed = df_skewed.withColumn("join_key", concat("original_key", lit("_"), "salt"))
```

## 🚨 The Incident: NULL スキュー
**事象**: JOINのキーに `NULL` が大量に含まれており、全ての `NULL` 行が一つの Executor に集まってしまい爆発した。
**解決策**: 結合に関係ない `NULL` は事前にフィルタして除外するか、ダミーのランダム値に置き換えてから JOIN します。
