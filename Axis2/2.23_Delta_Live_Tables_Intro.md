# [2.23] Delta Live Tables (DLT)：宣言型パイプラインの構築

「データパイプラインを1つのプロジェクト（ソースコード）として管理しよう」という考え方です。

### 1. SQL / Python による宣言
`CREATE TABLE ...` を書くのではなく、**「このテーブルは、あのテーブルから作られる」という『関係性』**だけを記述します。
```sql
-- DLT SQLの例
CREATE OR REFRESH STREAMING TABLE raw_orders
AS SELECT * FROM cloud_files("/raw/path", "csv")
```

### 2. DLTの強み
*   **依存関係の自動解決**: 「このテーブルを作る前に、あっちを更新しなきゃ」というのをDatabricksが自動判断します。
*   **パイプラインの可視化**: 画面上で、データがどう流れていくかのグラフ（DAG）が美しく表示されます。
*   **監視機能**: 処理件数や成功率のログが標準で蓄積されます。

---
**💡 アーキテクトの視点:**
従来の「Notebookを何個も繋げてジョブを組む」時代から、DLTのような「宣言型（コードが仕様）」の時代へ。これが2024年のデプロイの基本となります。
