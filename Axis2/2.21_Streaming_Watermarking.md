# [2.21] Streaming における Watermarking（遅延データの破棄）とステート管理

ストリーミング処理において、「遅れてやってきたデータ（Late Arrival）」をいつまで待つか、という問題です。

### 1. Watermarking (ウォーターマーキング)
「データの時刻（Event Time）から、現在は1時間経っている」と仮定し、それ以上古いデータが来たら **無視（Drop）** するという境界線のこと。
```python
# 10分以上の遅延データは無視して集計を締め切る
df.withWatermark("event_time", "10 minutes") \
  .groupBy(window("event_time", "5 minutes")) \
  .count()
```

### 2. なぜ待つのを止める必要があるのか？
Sparkは「まだ後からデータが来るかもしれない」と思って、中間的な集計結果（State）をメモリ上に保持し続けます。
**Watermarkを設定しないと、メモリがいつか爆発（OOM）します。**

---
**💡 アーキテクトの視点:**
リアルタイムの不確実性と、システムの安定性（メモリ消費）のトレードオフを決定するのがこの設計です。
「この分析において、1時間以上遅れてきたデータはゴミとして扱ってもよいか？」というビジネス要件の合意が不可欠です。
