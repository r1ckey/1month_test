# [1.17] Databricks REST APIによる各種自動化

Databricksの全ての操作は、UIを介さずに **「REST API」** を通じてプログラム（Python, PowerShell, cURL）から実行可能です。

## 1. APIの主な用途
*   **オートメーション**: コードをアップロードしてJobを自動作成〜実行。
*   **オーケストレーション**: 外部ツール（Azure Data Factory等）からのキック。
*   **モニタリング**: 実行中のクラスタの状態や設定情報をプログラムで取得。

## 2. 認証方法 (Personal Access Token / PAT)
APIを叩くには認証が必要です。
*   **PAT (Personal Access Token)**: ワークスペース上で生成するトークン。
*   **Service Principal Token**: 本番運用の場合は、個人ではなくサービスプリンシパルのトークンを渡して認証するのが定石です。

## 3. Pythonによる実行例 (`requests` ライブラリ)
```python
import requests

DOMAIN = 'adb-xxxxxxxxxxxxxx.x.azuredatabricks.net'
TOKEN = 'dapi_xxxxxxxxxxxxxxxxxxxx'

# クラスタ一覧を取得するAPI
response = requests.get(
    f"https://{DOMAIN}/api/2.0/clusters/list",
    headers={"Authorization": f"Bearer {TOKEN}"}
)

print(response.json())
```

> [!TIP]
> **API 2.1 vs 2.0**
> 現在は多くの機能が v2.1 に移行しています。最新ドキュメントを確認し、可能な限り新しいエンドポイントを使うようにしましょう。
