# [1.17] Databricks REST API 入門：プログラムによる自動化の基礎

「手動でポチポチ作業する」のを卒業し、コードで数千のワークスペースを操ります。

## 1. REST API の主な用途
- **大量クラスタの一括作成**: 100個のクラスタのポリシーを一括で書き換える。
- **CI/CD連携**: GitHub Actions から Notebook をデプロイする。
- **監視の自動化**: 特定のJobが失敗したら、外部の通報システムに情報を投げる。

## 2. 認証方式 (Authentication)
1.  **Personal Access Token (PAT)**: お手軽だが、個人に紐づくため退職時に無効化されるリスクあり。
2.  **Entra ID (Azure AD) トークン**: 実務ではこちらを推奨。SPN（サービスプリンシパル）でトークンを取得し、安全にAPIを叩きます。

## 3. Python (`requests`) による実行例
```python
import requests

token = "dapi-xxxxxx"
domain = "adb-xxxx.azuredatabricks.net"

# クラスタの一覧を取得する例
response = requests.get(
    f"https://{domain}/api/2.0/clusters/list",
    headers={"Authorization": f"Bearer {token}"}
)

for cluster in response.json().get('clusters', []):
    print(f"Cluster Name: {cluster['cluster_name']}")
```

## 🚨 The Incident: レート制限（Rate Limiting）
**事象**: ループを回してAPIを大量に（毎秒数百回）叩いたところ、`429 Too Many Requests` が返ってきて、ポータル全体も動作が重くなった。
**解決策**: APIにはレート制限があります。大規模な自動化では、APIのレスポンスを見ながら「スリープ（待機）」を挟むようにコーディングしてください。
