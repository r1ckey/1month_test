# [1.02] プライベートリンク（Front-end / Back-end）による完全閉域化

エンタープライズ企業において、「インターネット経由でDatabricksの管理画面にアクセスする」ことはセキュリティリスクと見なされます。ここで登場するのが **Azure Private Link** です。

## 1. 2つのプライベートリンク
Databricksには2方向の閉域化が存在します。

### A. Front-end Private Link (User-to-Workspace)
ユーザーのブラウザからDatabricksのワークスペースへのアクセスを、パブリックインターネットではなく、Azure内仮想ネットワーク経由に制限します。
*   **メリット**: 社内ネットワーク（VPN/ExpressRoute経由）からしか、Databricksに入れないようにできる。

### B. Back-end Private Link (Compute-to-Control Plane)
Workerノード（Compute Plane）から、Databricksの管理機能（Control Plane）への通信を閉域化します。
*   **メリット**: インターネットへのアウトバウンドを完全に遮断しても、クラスタが正常に動作し続ける。

## 2. 実装の注意点
Private Linkを導入する場合、以下の「プライベートDNSゾーン」の設定が必須となります。
*   `privatelink.azuredatabricks.net`

> [!WARNING]
> **パブリックアクセスの無効化**
> 物理的にPrivate Linkを通しても、ワークスペース設定で「Public Network Access」を `Disabled` にしない限り、インターネットからの入り口が開いたままになるため注意が必要です。

---
**💡 実務のヒント:**
Azure Private Linkはコストがかかります（Private Endpointごとに毎時料金が発生）。全環境に導入するか、本番環境のみにするかはコストバジェットとの相談になります。
