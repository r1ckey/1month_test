# [1.22] DBU (Databricks Unit) と クラウドコストモニタリング・Tag付け

Databricksの料金体系を理解することは、アーキテクトにとって必須のスキルです。

## 1. DBU (Databricks Unit) とは？
Databricks独自の計算リソース単位です。
*   **料金の合計**: **[Azure VM インフラ料金] + [Databricks DBU 料金]**
*   **変動要因**: クラスタのVMサイズ（ノード数）と、1時間あたりの稼働時間で決まります。

## 2. コスト削減の「Tag (タグ)」戦略
Azureのリソースに「ProjectName」や「Environment: Production」といったタグ（Tag）を付与することで、Azureのコスト分析（Cost Management）で正確な請求先を特定できます。

## 3. 実務でのタグ付け自動化
クラスタポリシー（前章）を使って、クラスタ作成時に特定のタグを強制（Required）させます。
```json
{
  "custom_tags.Project": { "type": "fixed", "value": "A-Project" },
  "custom_tags.CostCenter": { "type": "allowlist", "values": ["101", "102"] }
}
```

> [!IMPORTANT]
> **DBU料金の「Tier（ティア）」**
> 「Jobs（バッチ処理向け）」は安く、「All-Purpose（開発時のNotebook向け）」は高く設定されています。ジョブのスケジューリングには必ず Jobs クラスタ（Job Compute）を使いましょう。

---
**💡 運用設計のポイント:**
「DBU予算の週次モニタリング」を運用手順書に盛り込み、予算を超えそうな場合に自動通知を送る設計が、クライアントから高く評価されます。
