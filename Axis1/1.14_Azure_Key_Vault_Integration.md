データベースのパスワードやAPIキーをソースコード（Notebook）に直書きするのは厳禁です。

## 1. Secrets (シークレット) の仕組み
Databricksには「秘密情報」を隠して呼び出すための管理機能があります。
1.  **Secret Scope (シークレットスコープ)**: 秘密情報をまとめる入れ物。
2.  **Secret (シークレット)**: キーと値のペア。

## 2. Azure Key Vault (AKV) 連携
実務では「Databricks自体にパスワードを保存する」のではなく、「Azure Key Vault」と同期（バッキング）させるのが標準です。
*   **メリット**: Azure側で一元管理でき、ローテーション（パスワード更新）も容易。
*   **設定**: Databricksのシークレットスコープ作成時に、AKVの「DNS名」と「リソースID」を指定します。
