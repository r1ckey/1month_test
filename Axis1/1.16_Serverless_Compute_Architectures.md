# [1.16] Serverless SQL Warehouse と Jobs：管理工数ゼロの次世代コンピューティング

「インフラを意識しない」ことが、最も生産性を高める究極の形です。

## 1. サーバレス・コンピューティングとは？
通常のクラスタと異なり、**あらかじめ起動された共用リソースプール**を使用します。
- **インスタント起動**: 通常 3〜5分かかるクラスタ起動時間が、3〜10秒に短縮されます。
- **管理不要**: VNetの設計やパッチ適用、停止忘れなどを一切気にする必要がありません。
- **秒単位の課金**: 処理が終われば即座に課金が止まるため、極めて効率的です。

## 2. Serverless SQL Warehouse の実務活用
- **BIツール (Power BI / Tableau)**: 経営層がダッシュボードを開いたとき、クラスタ起動を待たせることなく即座にグラフを表示できます。
- **SQLノートブック**: 手動でのアドホックなクエリ実行に最適です。

## 3. Serverless Jobs のメリット
バッチ処理（Job）でもサーバレス化が進んでいます。
- 従来のJob用クラスタは「起動時間」にも費用がかかっていましたが、サーバレスJobは「実計算時間」のみの課金となるため、10分以内の短いジョブほどコスト削減効果が高くなります。

## 🚨 The Incident: 特定のライブラリが使えない
**事象**: サーバレスJobでPythonの特殊なライブラリ（地理情報系等）を使おうとしたが、インストールが制限されており動作しなかった。
**解決策**: 2024年のサーバレスはまだ環境（ランタイム）が固定されています。特殊なライブラリやOSレベルの設定が必要な場合は、引き続き「プロビジョニング済み（通常の）クラスタ」を使用してください。
