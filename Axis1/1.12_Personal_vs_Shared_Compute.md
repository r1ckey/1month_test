# [1.12] パーソナル・コンピュート vs 共有クラスタ（分離レベルとUser Isolation）

Databricksのクラスタには、使い勝手とセキュリティが異なる複数の「アクセスモード」があります。

## 1. パーソナル・コンピュート (Single User Access Mode)
特定の1人のユーザー専用のクラスタです。
*   **用途**: 個人でのアドホックな分析、開発。
*   **特徴**: 全てのライブラリのインストールや、Python/Scala/SQL全ての言語が使えます。
*   **欠点**: 複数人で同時に使う（同時実行）ことはできません。

## 2. 共有クラスタ (Shared Access Mode)
複数のユーザーが1つのクラスタを共同で使います。
*   **特徴**: **ユーザーレベルの分離 (User Isolation)** が働いています。
*   **セキュリティ**: Aさんの実行したコードがBさんの機密インメモリデータに触ることはできません。
*   **制約 (Unity Catalog環境下)**: Unity Catalogを有効にした共有モードでは、現在Scalaはサポートされておらず、**Python と SQL** のみが利用可能です。

> [!IMPORTANT]
> **共有モードの使い所**
> クラスタの起動時間を節約するために、開発チーム全体で1つの「常時起動（またはオートスケール）された共有クラスタ」を使い回す運用もあります（コスト削減に繋がります）。

---
**💡 チェックポイント:**
資格試験では「Unity Catalog環境で複数のユーザーが1つのクラスタを使う場合、どのアクセスモードを選択すべきか？」と問われます。答えは「Shared」です。
