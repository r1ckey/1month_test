# [1.15] Log Analytics と 診断設定 (Diagnostics) による監査ログ抽出

「誰がいつどのテーブルを見たか」「誰がこのクラスタを消したか」といった記録は、コンプライアンス上極めて重要です。

## 1. 診断設定 (Diagnostic Settings)
Azureポータル上のDatabricksワークスペースの画面から設定します。
*   **転送先**: **Log Analytics ワークスペース** (推奨) や ストレージアカウント。

## 2. ログの種類
*   **auditLogs**: ログイン記録、ファイルの読み書き、権限変更。
*   **notebookLogs**: Notebookで誰が何を実行したかのコマンド。
*   **clusterLogs**: クラスタの起動、終了、エラーメッセージ。

## 3. Kusto クエリ (KQL) による分析
Log Analytics上のデータをSQLに近い KQLで分析します。
```kql
// 特定のユーザーのクラスタ作成履歴を抽出する例
DatabricksAccounts
| where ActionName == "createCluster"
| where Identity == "target_user@example.com"
```

> [!IMPORTANT]
> **Unity Catalogの監査ログ**
> Unity Catalogを有効にしている場合、UC経由のアクセスログは「アカウントレベル」で管理されるため、個別のワークスペースの診断設定とは別に収集・分析する仕組みが必要になります。

---
**💡 運用設計のポイント:**
「異常なログイン」や「高額クラスタの連日起動」を検知して管理者にメールを送るアラート設計を、運用フェーズのタスクとして盛り込みます。
