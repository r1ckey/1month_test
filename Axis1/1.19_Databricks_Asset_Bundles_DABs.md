# [1.19] Databricks Asset Bundles (DABs) の概念と CLIセットアップ

Databricksにおける「最新のデプロイスタンダード」が **Databricks Asset Bundles (DABs)** です。

## 1. 従来のデプロイの課題
これまでは、Notebookのコード、Jobの設定、クラスタの設定をバラバラにデプロイする必要があり、非常に管理が煩雑でした。

## 2. DABs とは？
「コード」と「インフラ（Job/Workflow）」を一つの **「バンドル（束）」** として定義し、一括でデプロイ・バージョン管理する仕組みです。

## 3. Databricks CLI のセットアップ
DABsを利用するには、まずコマンドラインツール（CLI）をインストールし、ワークスペースに認証（認証プロファイル作成）する必要があります。
```bash
# プロファイルの作成 (ブラウザが開いて認証される)
databricks auth login --host https://adb-xxxxxx.azuredatabricks.net
```

## 4. プロジェクトの初期化
```bash
# 対話形式でプロジェクトの雛形が作成される
databricks bundle init
```

> [!TIP]
> **「Software Engineering」としてのデータ開発**
> DABsを導入することで、データパイプライン開発を、これまでの「適当なNotebookの切り貼り」から、「JavaやPythonアプリ開発のような、高度に成熟したソフトウェア開発サイクル」へと引き上げることができます。

---
**💡 アーキテクトの視点:**
Terraformは「インフラそのもの（VNetやWS）」の構築に向いており、DABsは「データ処理コードとJobスケジュール」の頻繁な更新に向いています。この2つを組み合わせて使うのが、2024年以降の最適解です。
