# [1.25] Azure Private DNS ゾーンとDatabricksワークスペースの名前解決

Private Linkを構築した際、「名前（URL）は分かるが、IPが分からない」という通信トラブルが多発します。

## 1. なぜ Private DNS が必要なのか？
通常、`adb-xxxxx.azuredatabricks.net` というURLはパブリックなIPを返します。これを「Azure内のプライベートなIP」として解決させるための看板が必要です。

## 2. 代表的なDNSゾーン
*   `privatelink.azuredatabricks.net`
*   `privatelink.blob.core.windows.net` (ADLS Gen2へのアクセス用)

## 3. リンク（DNS Link）の設定
作成した Private DNS ゾーンを、自身の **仮想ネットワーク (VNet)** にリンクさせなければなりません。これを忘れると、閉域網内蔵のVMから名前解決ができずに `Connection Timeout` エラーになります。

---
**💡 ネットワークトラブルシューティング:**
もし接続できない場合は、Workerノード（VM）にSSHログイン、またはNotebookから `%sh nslookup <workspace-url>` を実行してください。もしパブリックなIP（13.x.x.xなど）が返ってきたら、DNSの設定漏れです。10.x.x.xといった社内IPが返ってくれば正解です。
