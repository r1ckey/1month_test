# [1.13] Azure スポット VM（Azure Spot VMs）活用によるコスト 70-90% 削減術

「本番環境はフルプライス、開発環境は中古品（スポット）」
クラウドならではの節約テクニック。

## 1. Azure スポット VM とは？
Azureのデータセンターで余っている計算リソースを、格安で借りる仕組みです。
- **リスク**: Azure側でそのリソースが必要になった時、**事前通知から30秒で一方的に強制シャットダウン（Eviction）** されます。

## 2. Databricks での設定方法
クラスタ設定画面で「Spot Instances」にチェックを入れるだけです。
- **推奨設定**: Driverノードは「オンデマンド（通常価格）」にし、Workerノードだけを「スポット」にします。
- **なぜ？**: Driverが落ちるとクラスタ全体が死にますが、Workerが落ちる分には、Databricksが自動で代わりのWorkerを探して処理を再開してくれるからです。

## 3. 実務での適用判断
- **❌ 本番の夜間バッチ**: 朝までに終わらなければならない重要なJobにスポットは使いません。一回の強制終了でスケジュールが崩壊するからです。
- **⭕️ 開発・サンドボックス環境**: 途中で落ちても「リトライすればいい」環境では、コスト削減効果が絶大です。

---
## 🚨 The Incident: 全くクラスタが立ち上がらない！
**事象**: スポットVM設定でクラスタを起動しようとしたが、10分待っても起動しない。
**原因**: その特定の時刻に、そのリージョンでスポットVMの「空き」がゼロだった。
**解決策**: スポットVMの設定で「最大価格を無制限」にするか、一旦通常のVMに戻して起動させます。
