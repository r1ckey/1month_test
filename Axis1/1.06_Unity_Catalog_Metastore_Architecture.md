# [1.06] Unity Catalog (UC) の全体アーキテクチャ（MetastoreとCatalog）

Unity Catalogは、Databricks全体のガバナンスを管理するための「単一のガバナンス・ポイント」です。

## 1. 従来のメタデータ管理 (Hive Metastore) との違い
これまでは、ワークスペースごとにバラバラにメタデータ（テーブル定義）を管理していましたが、Unity Catalogにより「全リージョン、全ワークスペース共通のメタデータ」が実現しました。

## 2. Metastore (メタストア)
Unity Catalogの「脳」にあたる実体（コンテナ）です。
*   **特徴**: 1つのリージョンにつき1つ作成し、複数のDatabricksワークスペースを紐付けます。
*   **管理権限**: `Metastore Admin` という役割のユーザーがMetastore全体を統括します。

## 3. Catalog (カタログ)
Metastore内の最初の論理的な区分けです。
*   **実務上の分け方**: 
    1. プロジェクト・部門別 (例: `sales`, `marketing`, `hr`)
    2. 環境別 (例: `dev`, `stg`, `prod`)

> [!IMPORTANT]
> **MetastoreはADLS Gen2に紐付く**
> Unity Catalogを有効にするには、あらかじめ専用のADLS Gen2コンテナを用意し、Metastoreの作成時にそのURLを指定する必要があります。これが「カタログデータの物理的な保存先」となります。
