# [1.09] UCパーミッション：GRANT/REVOKE 構文とセキュリティ分離

Unity Catalogでのアクセス制御は、標準的なSQLのように `GRANT` 命令（および UI）で行います。

## 1. 主要な権限一覧

*   **USE CATALOG**: カタログを使用する権限。まずこれが無いと中身が見えません。
*   **USE SCHEMA**: スキーマを使用する権限。
*   **SELECT**: データを読み取る権限。
*   **MODIFY**: データの追加・更新・削除を行う権限。
*   **CREATE TABLE**: テーブルを作成する権限。

## 2. 実践的な GRANT 構文
```sql
-- 特定のユーザーグループに特定のカタログ内の全スキーマへの読み取り権限を与える
GRANT USE CATALOG ON CATALOG sales TO `data_scientists`;
GRANT USE SCHEMA ON SCHEMA sales.gold TO `data_scientists`;
GRANT SELECT ON SCHEMA sales.gold TO `data_scientists`;
```

## 3. 所有権 (Ownership)
テーブルを作成したユーザーは自動的に `Owner` となり、そのテーブルに対する全権限を持ちます。実務では退職対策として、所有権を個人から「グループ（例: `engineering_group`）」へ変更（ALTER）しておくのが定石です。

> [!WARNING]
> **「ALL PRIVILEGES」の乱用に注意**
> `GRANT ALL PRIVILEGES ...` は管理者以外には与えないのが基本です。

---
**💡 運用設計のポイント:**
権限管理は必ず「個々のユーザー」ではなく「Entra ID のグループ」に対して行うように運用設計書で規定してください。メンテナンスの手間が10倍変わります。
