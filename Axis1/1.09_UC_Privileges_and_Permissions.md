# [1.09] UCパーミッション：SQLによる権限制御 (GRANT / REVOKE)

Azureポータルを触ることなく、SQLだけで「誰にどのデータを見せるか」を完結させる。

## 1. 権限の種類（Privileges）
Unity Catalogでは以下の権限を組み合わせて制御します。
- `USAGE`: そのオブジェクト（カタログやスキーマ）を「通って良い」権限。
- `SELECT`: テーブルの読み取り。
- `MODIFY`: データの追加・更新・削除。
- `CREATE`: 新しいテーブルやスキーマの作成。
- `ALL PRIVILEGES`: 特権。

## 2. 権限継承（Inheritance）の仕組み
上位オブジェクトで与えた権限は、下位に引き継がれます。
- カタログレベルで `USAGE` を与えないと、その中のスキーマに `SELECT` があってもアクセスできません。
- **実務のコツ**: 「カタログには `USAGE` を全員に、特定のスキーマやテーブルには `SELECT` を特定のグループに」という絞り込みが標準です。

## 3. 実践的なSQLコマンド
```sql
-- 開発チーム(group_dev)に、売上カタログ(sales)の使用を許可する
GRANT USAGE ON CATALOG sales TO `group_dev`;

-- 売上明細テーブル(main.sales.details)の読み取りだけを許可する
GRANT SELECT ON TABLE main.sales.details TO `group_dev`;

-- 権限を確認する
SHOW GRANTS ON TABLE main.sales.details;
```

---
## 🚨 The Incident: 「権限があるのに見えない！」問題
**事象**: メンバーに `SELECT` 権限を与えたが、`Access Denied` エラーが出る。
**原因**: そのメンバーに、上位のカタログおよびスキーマに対する **`USAGE` 権限** を与え忘れている。
**教訓**: 下から順に権限をつけるのではなく、上（Catalog）から順番に「通れる道」を作ってあげることが重要です。
