# [3.03] ネットワーク仕様書設計：VNetのIPレンジ計算とSubnet/FW

データエンジニアと言えども、クラウド上の「番地（IP）」を設計できなければ構築は任されません。

### 1. VNet / Subnet サイジング
*   **計算方法**: 予定される最大クラスタ数、オートスケールの余裕、ワークスペースの増設可能性。
*   **例**: `/24` なら256個のIP。DatabricksのWorkerノード1台につき2つのIPを使用するため、128台までスケール可能。

### 2. 通信（FW）ポリシー
*   「どのサーバからのアクセスを許可（Allow）し、それ以外を拒否（Deny）するか」をマトリクスで記述します。
*   Azure FirewallやNSGのログから、想定外の通信が行われていないかを検証する基準となります。

---
**💡 ネットワーク設計の掟:**
IPアドレス案は一度決めると変更が困難（VNetを作り直すレベルの地獄）です。必ず **「10年後も溢れないサイズか？」** を自問自答した上で確定させてください。
