# [3.03] ネットワーク仕様書設計： Private Link 接続図の描き方

「繋がらない」のは、図が描けていないからです。

## 1. ネットワーク仕様書の必須項目
1.  **VNet / Subnet 一覧表**: CIDRレンジ、サブネットの用途、委任（Delegation）の有無。
2.  **NSG 許可リスト**: Inbound / Outbound のポート番号、送信元、送信先。
3.  **Private Endpoint マッピング**: どのリソースがどのプライベートIPを持つか。

## 2. 実務で「差が出る」ネットワーク構成図
単に線で繋ぐのではなく、以下の「境界線」を明確に描き込みます。
- **オンプレミス環境** (VPN / ExpressRoute の先)
- **Azure 管理環境** (Control Plane)
- **自社 VNet** (Compute Plane)
この 3つの境界をパケットがどう跨ぐかを矢印で描ければ、トラブル時の切り分けが 10倍 速くなります。

## 3. IP管理表の実例
Excel等で以下を管理します。
| 場所 | リソース名 | プライベートIP | 役割 |
| :--- | :--- | :--- | :--- |
| subnet-pe | pe-adb-frontend | 10.1.5.4 | UIアクセス用 |
| subnet-pe | pe-adls-blob | 10.1.5.10 | データアクセス用 |

## 🚨 The Incident: 重複したIPレンジ
**事象**: VNet Injection用に `10.0.0.0/16` を選んで構築したが、オンプレミス側のネットワーク担当から「そこは既に使っているレンジだ、VPN接続できない」と怒られ、全て作り直しになった。
**対策**: クラウド側のIPレンジを決める前に、必ず**全社インフラ担当**から「使って良いレンジ（ホワイトリスト）」を払い出してもらうステップを設計プロセスに組み込みます。
