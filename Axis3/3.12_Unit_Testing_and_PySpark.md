# [3.12] 【テスト仕様書②】単体テスト(UT)戦略：PySparkのpytest / CI組込

ソースコード単体が正しいか。プログラムによって自動で検証します。

### 1. 単体テスト (Unit Test)
*   **単位**: 関数またはひとつのモジュール単位。
*   **ツール**: Pythonでは `pytest` が標準。
*   **内容**: 計算ロジック、文字列操作、型チェック。

### 2. PySparkにおけるUTの難しさ
「Sparkの環境（SparkSession）」を起動しないと、DataFrameのテストができません。
これを解決するために、テスト時のみ軽量なローカルSparkを立ち上げる `pyspark-test` などのライブラリや `conftest.py` の活用が実務の知恵です。

---
**💡 運用の仕組み:**
GitHub ActionsなどのCIツールと連携し、「コードをマージするたびに、自動的に全ての単体テストが走り、1つでもエラーがあればマージを止める」というガードを敷くことが定石です。
