# [3.06] インターフェース（I/F）仕様：API vs ファイル（SFTP/Blob連携）

システムの外側（別のシステム）と、どうやってデータをやり取りするかを定義します。

### 1. I/Fの種類

*   **ファイル連携 (推奨)**: 相手のBlob StorageやSFTPサーバにファイルを置く。
    *   **メリット**: 大量データに強い。接続が一時的に切れても「再送」が容易。
*   **API連携**: REST API等でリアルタイムに一件ずつ送受信。
    *   **メリット**: 即時性が高い。

### 2. I/F仕様書の項目
*   **物理名称・ディレクトリ**: どこから何を読み、どこへ置くのか。
*   **授受タイミング**: 日次、リアルタイム、トリガー。
*   **ファイル拡張子/エンコーディング**: `.csv`? `.parquet`? `UTF-8`? `SJIS`（罠）?
*   **ヘッダー/フッター**: 有無とスキップ処理の要否。
*   **完了通知**: ファイルを置いた後に `_SUCCESS` ファイルを置くなどの「合図」の有無。

---
**💡 ネットワーク設計の掟:**
閉域網（Internal VNet）内でADLS連携をする場合、相手側のサブスクリプションを信頼（Managed Identityの許可）する設定を、この段階で合意しておく必要があります。
