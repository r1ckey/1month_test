# [3.10] バッチジョブフロー設計：依存関係の可視化と制御

「ただ並べる」のではなく、「順序」を構築する。

## 1. ジョブフローの 2つの型
1.  **時系列起動**: 深夜1時に A、2時に B、と時間をずらして起動。
    - **欠点**: Aが遅延すると、Bが不完全なデータを読み始める。
2.  **イベント / 依存関係起動 (Workflow)**: Aが終わったら Bを起動する。
    - **実務**: Databricks Workflows や Azure Data Factory (ADF) を使い、こちらを採用するのが標準です。

## 2. 同時実行の制御 (Parallel vs Sequential)
- **並列実行**: A, B, C のマスター取り込みを同時に行う（時短）。
- **逐次実行**: 全てのマスターが終わってから、D の集計を行う。
この「合流待ち（Join）」の設計を、Workflows の GUI 上で定義します。

## 3. 条件分岐（Conditionals）
- 「もし A が失敗したら、リカバリ用のジョブ C を動かす」
- 「もし A が成功し、かつ取得件数が 0件 でなければ B を動かす」

---
## 🚨 The Incident: ジョブの「多重起動」パニック
**事象**: 1時間おきに実行される Job A が何らかの理由で詰まり、30分経過しても終わらなかった。そこへ次の時間の Job A が起動してしまい、同じファイルを奪い合って両方クラッシュした。
**対策**: ジョブの設定で **「最大同時実行数 (Maximum Concurrent Runs)」** を `1` に制限し、前のジョブが終わるまで次が立ち上がらないように制御します。
