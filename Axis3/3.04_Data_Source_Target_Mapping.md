# [3.04] データ連携設計：ソース・ターゲットマッピングの書き方（Excel仕様）

開発者が最も多くの時間を費やす、かつ最も「地味に重要な」ドキュメントです。

### 1. マッピング表の項目
*   **Source**: 列名、型、NULL許可、サンプルの値。
*   **Target**: 移行後の列名（論理/物理）、型、ビジネスルール。
*   **変換ロジック**: 「`TRIM` する」「日付加算を行う」「マスタから取得」などの具体的な指示。

### 2. 実務の罠
「型変換」による情報の欠落に注意します。
*   例: decimal(20, 2) を float に変換すると、端数の計算結果が微妙にズレる恐れがある。これらをあらかじめマッピング段階で言語化（意思決定）しておきます。

---
**💡 ドキュメント作成術:**
Excelでこの表を作るときは、「開発者がコピペでそのままSQLの `CAST` や `SELECT` 文を書ける」レベルに解像度を上げてください。開発速度が2倍になります。
